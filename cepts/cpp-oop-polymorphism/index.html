    C++ 面向对象：继承、多态与虚函数¶

    [!note] C++ 面向对象的三个支柱——继承表达 is-a 关系，虚函数提供运行时多态，对象布局决定调用成本。三者不是独立的知识点，而是一条从设计意图到机器码的完整链路。

    1. 继承：表达类型关系，不是复用手段¶

    三种继承方式¶

    继承方式 基类 public 成员在派生类中 基类 protected 成员 表达语义
    public public protected is-a：派生类是一种基类
    protected protected protected 实现继承：外部不可见
    private private private 基于...实现：不是 is-a 
    // ✅ public 继承 = is-a
class Dog : public Animal {};   // Dog 是一种 Animal

// ❌ 为复用代码而继承
class Stack : private std::vector<int> {};  // Stack 不是 vector
//  → 应该用组合：class Stack { std::vector<int> data_; };

    继承 vs 组合¶

    维度 继承 组合
    关系 is-a（语义从属） has-a（拥有/使用）
    耦合度 高——派生类依赖基类实现 低——只依赖接口
    运行时多态 ✅ 虚函数 ❌ 需要 wrapper
    复用方式 继承基类成员 委托给成员对象

    "组合优于继承"的意思是：如果只是复用实现，用组合。如果确实需要 is-a + 运行时多态，用继承。

    2. 多态：静态 vs 动态¶

    静态多态（编译期） 动态多态（运行期）
    机制 模板、重载、CRTP 虚函数 + 继承
    分派时机 编译期确定 运行时查虚表
    性能 可内联，零开销 间接跳转 ~5-10 cycles
    扩展方式 新增类型需重新编译 新增派生类无需改调用方
    典型场景 std::sort、std::vector<T> 插件系统、策略模式

    动态多态的三个成立条件¶

    // ① 基类有虚函数
class Shape { public: virtual void draw() = 0; };

// ② 派生类重写
class Circle : public Shape { public: void draw() override { /*...*/ } };

// ③ 通过基类指针/引用调用
void render(Shape& s) { s.draw(); }  // 运行时确定调用 Circle::draw

    多态失效的常见场景¶

    Circle c;
Shape s = c;     // ❌ 对象切片——s 是纯 Shape，Circle 部分丢失
s.draw();        // 调用 Shape::draw（如果非纯虚）

Shape& ref = c;  // ✅ 引用——不发生切片
ref.draw();      // 调用 Circle::draw

    3. 虚函数底层：vptr + vtable¶

    对象内存布局：
┌──────────────┐
│ vptr         │ → 指向虚函数表
│ 成员变量1    │
│ 成员变量2    │
└──────────────┘

虚函数表 (vtable)：
┌──────────────────┐
│ &Circle::draw    │  ← 槽位0（被派生类覆盖）
│ &Shape::area     │  ← 槽位1（未被覆盖，保留基类版本）
│ &Circle::~Circle │  ← 槽位2（虚析构）
└──────────────────┘

    调用链：s.draw() → 取 s 的 vptr → 查 vtable[0] → 跳转 Circle::draw

    虚调用的成本¶

    • 一次间接寻址（vptr → vtable → 函数）
    • 阻止内联（编译器不知道运行时类型）
    • 对分支预测和指令缓存不友好

    性能敏感的调用路径上避免不必要的虚函数，但不要为了"省几纳秒"牺牲清晰的抽象。

    虚析构函数¶

    class Base {
public:
    virtual ~Base() = default;  // ← 多态基类必须虚析构
};

Base* p = new Derived;
delete p;  // 如果 ~Base() 非虚 → 只调用 ~Base() → 派生资源泄漏

    4. 多继承与指针调整¶

    struct Left  { virtual void left();  int a; };
struct Right { virtual void right(); int b; };
struct Child : Left, Right {};

Child c;
Left*  lp = &c;   // 指向 Child 的 Left 子对象 —— 地址 = &c
Right* rp = &c;   // 指向 Child 的 Right 子对象 —— 地址 = &c + sizeof(Left)
// lp != rp ！

    多继承引入了this 指针调整：当通过 Right* 调用虚函数时，编译器需要将 this 调整回 Child 对象的起始地址。这是虚表实现中 thunk 代码的作用。
