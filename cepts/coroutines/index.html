    C++20 协程（Coroutines）¶

    [!note] C++20 协程提供的不是"开箱即用的异步框架"，而是语言级的状态机生成能力——编译器把协程函数改写为挂起/恢复的状态机，具体的调度和执行策略由库实现。

    1. 协程 ≠ 线程 ≠ 异步¶

    线程 协程
    调度者 操作系统内核 协程调度器（库层面）
    切换成本 内核态上下文切换（~μs 级） 用户态函数调用级（~ns 级）
    并发 真正并行（多核） 单线程内交替执行
    适用 CPU 密集型 IO 密集型、异步流程组织

    协程解决的是控制流组织问题——让你用同步风格的代码表达异步流程，而非回调地狱或状态机。

    2. 核心机制¶

    编译器视角：状态机生成¶

    // 你写的：
Task<int> compute() {
    int a = co_await read_async("a.txt");    // 挂起点1
    int b = co_await read_async("b.txt");    // 挂起点2
    co_return a + b;
}

// 编译器生成的状态机（简化）：
//  状态0: 调用 read_async("a.txt")，挂起
//  状态1: a = 恢复值，调用 read_async("b.txt")，挂起
//  状态2: b = 恢复值，返回 a+b

    协程帧（Coroutine Frame）¶

    编译器在堆上分配一块内存保存： - promise_type 实例 - 所有跨越挂起点的局部变量 - 当前状态编号 - 协程句柄（std::coroutine_handle）

    co_await 三步协议¶

    // 当遇到 co_await expr 时：
auto&& value = expr;

// ① await_ready() → true? 直接跳过挂起
if (!value.await_ready()) {
    // ② await_suspend(handle) → 挂起，保存协程句柄
    value.await_suspend(coroutine_handle);
    // ... 协程暂停在这里 ...
}

// ③ await_resume() → 恢复，返回结果
auto result = value.await_resume();

    三个关键字¶

    关键字 作用 关键约束
    co_await 挂起或恢复当前协程 操作数必须实现 awaiter 接口
    co_return 结束协程并设置返回值 调用 promise_type::return_value()
    co_yield 挂起并产出一个值 等价于 co_await promise.yield_value(expr)

    3. 典型应用¶

    生成器（Generator）¶

    Generator<int> range(int from, int to) {
    for (int i = from; i < to; ++i)
        co_yield i;
}
// 使用: for (int v : range(0, 10)) ...

    异步 IO¶

    Task<std::string> fetchUrl(const std::string& url) {
    auto socket = co_await connect_async(url);
    auto data = co_await read_async(socket);
    co_return data;
}

    4. C++20 协程的局限¶

    C++20 只提供了语言级机制，没有提供： - 标准生成器类型（C++23 才有了 std::generator） - 标准任务类型 - 标准异步 IO 调度器 - 标准协程线程池

    这意味着目前需要第三方库（如 cppcoro、libunifex）或自己实现基础设施。

    5. 关联概念¶

    • future-promise-async — std::async 是协程之前的异步方案，协程可以更优雅地组织多个异步操作
    • thread-pool — 协程调度器可以基于线程池实现
    • move-semantics — 协程帧中的局部变量支持移动语义以降低内存开销
    • lock-free-and-memory-model — 高性能协程调度器常用无锁队列
