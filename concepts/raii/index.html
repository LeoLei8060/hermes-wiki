    RAII（资源获取即初始化）¶

    [!note] RAII 是现代 C++ 资源管理的核心范式——把资源的获取与释放绑定到对象生命周期，利用确定性析构保证资源自动回收。

    1. 定义与原理¶

    RAII（Resource Acquisition Is Initialization）的核心不是"用智能指针"，而是让对象的生命周期管理资源的生命周期：

    • 构造 = 获取资源：对象构造成功意味着资源已就绪
    • 析构 = 释放资源：离开作用域时析构函数自动执行，无论退出路径是正常 return 还是异常展开
    • 异常安全：栈展开（stack unwinding）保证已构造的对象会被析构，不需要在每个分支写清理代码

    RAII 管理的资源远不止内存：文件句柄、互斥锁、socket、数据库事务、临时状态变更——任何需要"获取-使用-释放"成对操作的资源都适用。

    2. 底层机制¶

    RAII 依赖 C++ 的两项核心保证：

    1. 确定性析构：自动对象离开作用域时析构函数立即被调用（不像 GC 语言需要等未来某个不确定的时间点）
    2. 异常展开保序：异常抛出后，从 throw 点到 catch 点之间所有已构造的局部对象会按构造的逆序析构

    这两条合在一起意味着：你永远不需要为"如果中间抛异常怎么办"单独写清理代码。

    3. 典型应用¶

    场景 RAII 封装 管理的资源
    互斥锁 std::lock_guard / std::unique_lock 锁的加/解锁
    动态内存 std::unique_ptr / std::shared_ptr 堆内存
    文件操作 std::fstream（析构时关闭） 文件句柄
    数据库事务 自定义 TransactionGuard 事务的 commit/rollback
    状态回滚 ScopeGuard / finally 模式 临时状态恢复

    4. 常见误区¶

    编码陷阱¶

    • 析构函数抛异常：如果析构函数在异常展开期间又抛出异常，std::terminate 会被调用。析构函数应尽最大努力不抛异常。
    • 所有权模糊：一个资源被多个对象认为"属于自己"，最终导致 double-free 或泄漏。每个资源应只有一个明确的 RAII 所有者。
    • 资源获取分散在多个步骤：构造失败时可能部分资源未获取，析构时需要判断"哪些资源实际已持有"，增加复杂度。

    概念误区¶

    • "RAII 就是智能指针"——RAII 是通用范式，智能指针只是一种应用
    • "析构函数一定会被调用"——对于 new 出来的对象，如果没有 delete 或被智能指针管理，析构不会自动发生

    5. 关联概念¶

    • lock-guard-vs-unique-lock — lock_guard 和 unique_lock 是 RAII 在锁管理上的两种应用
    • [[smart-pointers]] — 智能指针将 RAII 应用于动态内存
    • move-semantics — 移动语义改变了 RAII 对象的所有权转移方式
    • exception-safety — RAII 是实现异常安全的基础手段

    6. 面试要点¶

    • 核心定义："把资源获取和释放绑定到对象生命周期，利用构造与析构保证自动化管理"
    • 异常安全是关键：即使抛异常，栈展开也会触发析构——这是 RAII 相比手动 try-finally 的核心优势
    • 举例要广泛：不只举 unique_ptr，还要举 lock_guard、文件句柄、事务回滚
    • 析构不抛异常是常见追问点
