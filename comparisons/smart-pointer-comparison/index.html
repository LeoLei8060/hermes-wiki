    智能指针对比：unique_ptr / shared_ptr / weak_ptr¶

    [!note] 智能指针的价值不在"自动 delete"，而在通过类型系统表达所有权语义。三种指针对应三种完全不同的所有权模型。

    对比维度¶

    维度 unique_ptr shared_ptr weak_ptr
    所有权 独占所有权，不可拷贝 共享所有权，引用计数 无所有权（观察者）
    拷贝 禁止 允许（引用计数+1） 允许
    移动 允许（转移所有权） 允许（引用计数不变） 允许
    资源释放时机 unique_ptr 析构时 最后一个 shared_ptr 析构时 不释放资源
    运行时开销 几乎为 0（等同于裸指针 + 删除器） 两次原子操作（增减引用计数）+ 控制块内存 lock() 一次原子读
    内存占用 sizeof(T*) + sizeof(删除器) 2× sizeof(T*)（对象指针 + 控制块指针） 2× sizeof(T*)
    循环引用 不会 会——引用计数永远不归零 不会——用于打破循环
    配合数组 unique_ptr<T[]> 支持 [] shared_ptr<T[]> C++17 起支持 无数组版本
    典型场景 工厂函数返回值、容器元素、PIMPL 共享缓存、多线程共享状态 观察者、打破循环引用、缓存观察
    线程安全 访问对象本身不保证 引用计数操作线程安全，对象本身不保证 lock() 线程安全

    内部布局¶

    unique_ptr<int>
┌─────────────┐
│ int* ptr     │ → 指向堆上的 int
│ (删除器可能  │
│  空基类优化)  │
└─────────────┘

shared_ptr<int>
┌──────────────┐     ┌──────────────────┐
│ int* obj_ptr  │ →  │ int (堆对象)      │
│ ctrl* ctrl   │ →  │ 控制块:            │
└──────────────┘     │  strong_count: 2  │
                     │  weak_count: 1    │
                     │  deleter          │
                     │  allocator        │
                     └──────────────────┘

    为什么用 make_shared / make_unique？¶

    // ❌ 两次堆分配 + 可能的内存碎片
auto sp = std::shared_ptr<int>(new int(42));

// ✅ 单次分配：对象和控制块紧邻
auto sp = std::make_shared<int>(42);

// ❌ 不安全：如果构造函数抛异常，new 的内存泄漏
foo(std::unique_ptr<Bar>(new Bar()), std::unique_ptr<Baz>(new Baz()));

// ✅ 安全：make_unique 内部分配，异常不会泄漏
foo(std::make_unique<Bar>(), std::make_unique<Baz>());

    循环引用与 weak_ptr¶

    struct Node {
    std::shared_ptr<Node> next;  // 所有权方向
    std::weak_ptr<Node> prev;    // 观察方向——打破循环
};

// 如果 prev 也用 shared_ptr:
// A.next → B, B.prev → A → 两者互相持有 → 永不析构

    weak_ptr::lock() 安全地"尝试提升"为 shared_ptr： - 对象仍存活 → 返回有效 shared_ptr（引用计数 +1） - 对象已销毁 → 返回空 shared_ptr

    选型决策树¶

    需要共享所有权？ → 是 → 可能有循环？ → 是 → 一个方向用 weak_ptr
                    │                 └ 否 → shared_ptr
                    └ 否 → unique_ptr（默认首选）

函数参数需要传递智能指针？ → 函数需要参与所有权？
  ├ 需要共享 → const shared_ptr<T>&（不常见）
  ├ 需要转移所有权 → unique_ptr<T>（值传递）
  └ 不需要 → T& 或 T*（裸指针/引用观察）

    关联概念¶

    • raii — 三种智能指针都是 RAII 的具体应用
    • move-semantics — unique_ptr 只支持移动语义，通过移动转移所有权
    • lock-free-and-memory-model — 原子引用计数在无锁编程中是常用模式
    • thread-pool — 线程池可用 shared_ptr 管理共享资源、weak_ptr 做安全观察
