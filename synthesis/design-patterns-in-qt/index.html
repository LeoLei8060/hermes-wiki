    设计模式在 Qt 中的体现¶

    [!note] Qt 的很多 API 设计本身就是经典设计模式的工程化落地——信号槽是观察者、QSortFilterProxyModel 是策略、QUndoCommand 是命令。面试官问"你用过哪些设计模式"时，从 Qt 框架中找答案最有说服力。

    1. 观察者模式 → Qt 信号槽¶

    经典观察者的问题¶

    // 原始观察者：手动管理订阅列表，生命周期易出错
class Subject { std::vector<Observer*> observers_; };
// 忘记 unsubscribe → 悬空指针 → 崩溃

    Qt 信号槽的工程化增强¶

    经典观察者 Qt 信号槽
    手动维护订阅列表 connect() 自动注册
    忘记取消订阅 → 悬空回调 任意一端 QObject 析构时自动断开
    通知总是在调用者线程 支持 5 种连接类型（同步/异步/阻塞）
    一对一或一对多 一对多 + 多对一 + 信号连信号
    类型不安全（void*） 编译期类型检查（新式 connect） 
    // 信号槽 = 观察者模式 + 元对象系统 + 线程安全 + 自动生命周期管理
connect(sensor, &Sensor::valueChanged,
        panel,  &Panel::refreshValue);
// sensor 析构或 panel 析构 → 自动断开，无需手动 unsubscribe

    面试话术："Qt 信号槽是观察者模式的工程化实现。它在经典观察者的基础上增加了元对象反射、线程安全的连接类型选择和自动的生命周期管理。"

    2. 策略模式 → QSortFilterProxyModel¶

    // 策略接口 = QSortFilterProxyModel 的虚函数
class MyFilterProxy : public QSortFilterProxyModel {
protected:
    // 策略1：过滤条件（可运行时替换）
    bool filterAcceptsRow(int row, const QModelIndex& parent) const override {
        auto idx = sourceModel()->index(row, 0, parent);
        return idx.data().toString().contains(filterText_);
    }
    // 策略2：排序比较（可运行时替换）
    bool lessThan(const QModelIndex& left, const QModelIndex& right) const override {
        return left.data().toInt() < right.data().toInt();
    }
};

    通过继承重写 filterAcceptsRow()/lessThan()，为同一份源数据提供不同的展示策略——不修改源 Model，不修改 View。这是策略模式"算法可替换"思想在 Qt 中的直接应用。

    3. 命令模式 → QUndoCommand¶

    Qt 的 Undo 框架是命令模式的教科书级实现：

    class AddCommand : public QUndoCommand {
public:
    AddCommand(Model* model, const Data& data)
        : model_(model), data_(data) {
        setText("Add item");  // 显示在 Undo/Redo 菜单中
    }
    void redo() override { model_->add(data_); }
    void undo() override { model_->remove(data_); }
private:
    Model* model_;
    Data data_;  // 保存撤销所需的完整状态
};

// 使用
auto* undoStack = new QUndoStack(this);
undoStack->push(new AddCommand(model, data));
// Undo: Ctrl+Z → undoStack->undo()
// Redo: Ctrl+Y → undoStack->redo()

    命令对象化 → 可排队、可撤销、可重放、可序列化。Qt 做了直接可用的封装。

    4. Qt 框架中的其他模式映射¶

    设计模式 Qt 中的体现 位置
    单例 QCoreApplication::instance() 全局应用对象
    工厂方法 QPluginLoader + 插件接口 插件系统
    适配器 QAbstractItemModel 适配任意数据源到 View 模型视图
    装饰器 QSortFilterProxyModel 包装源 Model 代理模型
    模板方法 QThread::run() — 框架定义骨架，子类填充 线程
    外观 QProcess 封装平台相关的进程管理 系统接口
    桥接 Qt 的平台抽象层（Windows/Linux/macOS 后端） 跨平台

    5. SOLID 原则在 Qt 桌面应用中的实践¶

    原则 Qt 开发中的体现
    S 单一职责 Worker 类只管业务，QThread 只管线程生命周期
    O 开闭原则 通过插件接口扩展功能，不修改主程序
    L 里氏替换 QStyledItemDelegate 可替代 QAbstractItemDelegate 的所有使用场景
    I 接口隔离 QAbstractListModel / QAbstractTableModel / QAbstractItemModel 按需继承
    D 依赖倒置 高层模块依赖 IExportPlugin 接口，不依赖具体的 PdfExportPlugin
