    面向对象设计原则¶

    [!note] SOLID 不是背诵题——它是一套判断代码为什么难改的检查表。违反某条原则的典型症状是：改一个地方需要改 N 个文件。

    SOLID 五条原则¶

    S — 单一职责 (SRP)¶

    一个类只应该有一个引起它变化的原因。

    // ❌ 一个类做了三件事：数据、格式化、存储
class Report {
    void loadData();
    void formatHtml();
    void saveToFile();
};

// ✅ 拆成三个类
class DataLoader { void load(); };
class Formatter   { std::string format(const Data&); };
class FileSaver   { void save(const std::string&); };

    判断标准：如果你描述这个类时用了"和"字——它可能做了太多事。

    O — 开闭原则 (OCP)¶

    对扩展开放，对修改关闭。

    新增功能时应该添加新代码，而不是修改已有的稳定代码。

    // ❌ 每加一个新格式就要改 if-else
void export(Format f) {
    if (f == PDF) { /*...*/ }
    else if (f == CSV) { /*...*/ }
    // 新增格式 → 修改这个函数
}

// ✅ 新增格式 = 新增类
class Exporter { virtual void export() = 0; };
class PdfExporter : public Exporter { /*...*/ };

    L — 里氏替换 (LSP)¶

    子类必须能替换基类而不破坏程序正确性。

    class Rectangle {
public:
    virtual void setWidth(int w) { width_ = w; }
    virtual void setHeight(int h) { height_ = h; }
};
class Square : public Rectangle {
public:
    void setWidth(int w) override { width_ = height_ = w; }
    void setHeight(int h) override { width_ = height_ = h; }
};
// 问题：Square::setWidth() 会改变 height → 违反调用方的预期

    LSP 的本质：子类不能强化基类的前置条件，不能弱化后置条件。

    I — 接口隔离 (ISP)¶

    不应强迫客户依赖它们不用的接口。

    // ❌ 胖接口
class Worker {
    virtual void code() = 0;
    virtual void manage() = 0;  // 普通程序员用不到
    virtual void report() = 0;  // 普通程序员用不到
};

// ✅ 按角色拆分
class Coder     { virtual void code() = 0; };
class Manager   { virtual void manage() = 0; };

    Qt 中的应用：QAbstractListModel / QAbstractTableModel / QAbstractItemModel 按需继承。

    D — 依赖倒置 (DIP)¶

    高层模块不应依赖低层模块，二者都应依赖抽象。

    // ❌ 高层直接依赖具体实现
class OrderService {
    MySqlDatabase db_;  // 换数据库 = 改 OrderService
};

// ✅ 依赖抽象接口
class OrderService {
    IDatabase& db_;     // 换数据库 = 注入新实现，OrderService 不变
};

    Qt 中的应用：插件系统的接口声明（IExportPlugin）——主程序依赖接口，不依赖具体插件。

    组合优于继承¶

    问题：继承的隐性耦合¶

    class Bird {
public:
    virtual void fly() { /*...*/ }
};
class Penguin : public Bird {
    void fly() override { throw std::runtime_error("can't fly"); }
    // 违反了 LSP——企鹅不能替换 Bird
};

    解决：用组合隔离能力¶

    class FlyBehavior {
public:
    virtual void fly() = 0;
};
class CanFly : public FlyBehavior {
    void fly() override { /*...*/ }
};
class CannotFly : public FlyBehavior {
    void fly() override { /* 空操作或报错 */ }
};

class Bird {
    std::unique_ptr<FlyBehavior> flyBehavior_;
public:
    void fly() { flyBehavior_->fly(); }
};
// Penguin = Bird + CannotFly —— 没有违反任何原则

    选择标准¶

    用继承当 用组合当
    存在 is-a 语义关系 只是想复用实现
    需要运行时多态（虚函数） 需要运行时切换行为
    派生类是基类的特化 派生类只是"恰好有一部分代码相同"
