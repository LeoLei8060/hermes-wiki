    Qt 工程部署与跨平台发布¶

    [!note] Qt 程序"在自己机器上能跑"只是第一步。"在用户机器上能跑"涉及运行时依赖收集、平台差异适配、安装器制作和自动化验证——这四件事构成了工程级别的部署能力。

    1. 部署产物清单¶

    一个 Qt 桌面应用的完整发布包不止一个 .exe：

    产物 说明 平台差异
    可执行文件 主程序 Windows .exe / Linux ELF / macOS .app
    Qt 运行时库 Qt6Core.dll 等 windeployqt 自动收集
    平台插件 platforms/qwindows.dll 每个平台一个，是 QPA 的底层
    图像格式插件 imageformats/qjpeg.dll 支持 JPEG/PNG/GIF 等
    QML 模块 如果用 QML qml/ 目录树
    翻译文件 .qm 文件 translations/
    资源文件 图片、配置、证书 可打包进 .exe (QRC) 或外部文件
    VC++ Runtime Windows 专属 vcredist 或静态链接
    安装器 用户安装体验 NSIS / WiX (Win) / DMG (Mac) / AppImage (Linux)

    2. 平台部署工具链¶

    # Windows
windeployqt --release --qmldir ./qml ./MyApp.exe
# 自动复制：Qt DLL、平台插件、图像插件、QML 模块、翻译文件

# macOS  
macdeployqt MyApp.app -dmg
# 自动复制 Qt 框架到 .app bundle + 修改 rpath + 做代码签名

# Linux
linuxdeployqt MyApp -appimage
# 打包为独立运行的 AppImage

    windeployqt 的原理¶

    1. 用 dumpbin 分析 .exe 的导入表 → 找到需要的 Qt DLL
    2. 复制 DLL 到发布目录
    3. 同时复制 platforms/、imageformats/ 等插件目录
    4. 不会复制 VC++ Runtime → 需要单独处理（vcredist 或静态链接 /MT）

    在干净环境验证¶

    # Docker / VM 中：Windows Sandbox 或干净虚拟机
# 只安装操作系统，不安装 Qt/Visual Studio
# 复制发布包 → 运行
# 缺少的 DLL 会立即暴露（Event Viewer 或 Dependency Walker）

    3. 跨平台差异清单¶

    差异点 Windows Linux macOS
    路径分隔符 \ / /
    数据目录 %APPDATA% $XDG_DATA_HOME 或 ~/.local/share ~/Library/Application Support
    可执行文件扩展名 .exe 无 .app 目录
    动态库扩展名 .dll .so .dylib
    平台插件 qwindows.dll libqxcb.so libqcocoa.dylib
    高 DPI 整倍缩放为主 分数缩放常见 Retina 2x
    字体渲染 ClearType FreeType Core Text
    安装器 NSIS / WiX / MSI AppImage / Flatpak / deb DMG / PKG

    条件编译与运行时检测¶

    // 编译期
#ifdef Q_OS_WIN
    #include <windows.h>
#endif

// 运行时：用 Qt 抽象优于条件编译
QString dataDir = QStandardPaths::writableLocation(
    QStandardPaths::AppDataLocation);  // 自动适配平台

// 处理平台特定的字体回退
QFont font("Microsoft YaHei");  // Windows 有
// 在其他平台自动回退到系统默认字体

    4. 可重复的发布流水线¶

    Git Tag → CI Build (Windows + Linux + macOS 矩阵)
→ 各平台 deploy 工具 → 安装器打包 → 干净环境冒烟测试
→ 上传发布 → 自动更新检查

    手工部署的三大反模式： - 在自己开发机上随便拷几个 DLL 就发出去 - 发布后从不验证 → 用户报"缺少 xxx.dll"才知道 - 每次发布的目录结构和文件名不一致

    5. 关联概念¶

    • abi-compatibility — 部署时不同模块（主程序、插件）的 ABI 必须兼容
    • qt-plugin-system — 插件的查找路径在不同平台上不同
    • qt-ui-development — 自定义控件的字体/DPI 适配需要考虑跨平台
    • optimization-methodology — 发布前做性能验证，确保优化在目标平台上生效
